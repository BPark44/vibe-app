Security Audit Report

vibe-app · Generated June 9, 2026 · 6 parallel audit domains

Audits covered authentication, API layer, dependencies, input validation, infrastructure/config, and secrets/data exposure.

Overall Risk: Medium–High until RLS and redirect validation are fixed

Executive Summary

The application has a sound application-layer auth model (Supabase SSR cookies, getUser() validation, protectedProcedure, user-scoped queries). The main risks are defense-in-depth gaps: missing Postgres RLS, privileged DATABASE_URL access, open redirects in auth flows, no security headers, and dependency advisories clustered around prisma-kysely.

Critical prisma/migrations/20260607061911_db_init/migration.sql:1-14 profiles table has no Row Level Security. With NEXT_PUBLIC_SUPABASE_ANON_KEY exposed to the client, Supabase PostgREST may allow direct table access, bypassing tRPC auth. Enable RLS on profiles; add policies scoped to auth.uid(). High src/server/db/index.ts:10-27 Kysely uses DATABASE_URL (privileged Postgres role), bypassing RLS. All authorization is application-only. Use least-privilege DB role, or user-scoped Supabase client with RLS. Keep app-layer checks as secondary.
High src/app/login/login-form.tsx:14,39 Open redirect: redirectTo from query string passed to router.push() without validation. Allow only same-origin relative paths (^/[a-zA-Z0-9/_-]*$); reject //, http:, @. Default to /dashboard.
High src/app/auth/callback/route.ts:14,28 Open redirect: next concatenated as ${origin}${next}. @evil.com tricks can redirect off-site. Validate next as safe relative path; use new URL(next, origin) and verify url.origin === origin.
High src/app/api/trpc/[trpc]/route.ts:6-14 No rate limiting on tRPC endpoint. Public profile.list and mutations can be scraped or abused. Add per-IP/user rate limits (Upstash, Vercel KV, edge middleware).
High next.config.ts:3-5 No security headers (CSP, HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy). Add headers() in next.config.ts or platform config.
High package.json / pnpm-lock.yaml 3 High dev dependency advisories via prisma-kysely@1.8.0 → @prisma/internals@5.3.1 (cross-spawn, lodash, tmp). Upgrade prisma-kysely to 3.x (aligned with Prisma 6), or add pnpm.overrides.
High pnpm-lock.yaml → postcss@8.4.31 Moderate XSS in CSS stringify (GHSA-qx2v-qp2m-jg93). Only prod-runtime advisory (next@16.2.6 → postcss). Override postcss@>=8.5.10 until Next bumps it; validate next build.
