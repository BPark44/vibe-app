# vibe-app

A type-safe Next.js starter wired end-to-end:

| Concern        | Tool                                                          |
| -------------- | ------------------------------------------------------------- |
| Framework      | **Next.js** (App Router, TypeScript)                          |
| Styling        | **Tailwind CSS** v4                                           |
| API layer      | **tRPC** v11 (+ TanStack React Query)                         |
| Validation     | **Zod** (input schemas on every router)                      |
| Schema         | **Prisma** (source of truth + migrations)                    |
| DB queries     | **Kysely** (typed SQL, types generated from the Prisma schema)|
| Auth           | **Supabase Auth** (`@supabase/ssr`, cookie-based sessions)   |
| Database       | **Supabase Postgres** (queried directly via Kysely)          |

> **How Prisma + Kysely fit together:** Prisma owns the schema and runs
> migrations. The [`prisma-kysely`](https://github.com/valtyr/prisma-kysely)
> generator turns `prisma/schema.prisma` into Kysely TypeScript types
> (`src/server/db/types.ts`). At runtime the app **never uses Prisma Client** —
> all queries go through Kysely against the Supabase Postgres connection.

## Getting started

### 1. Configure environment

Copy the example env file and fill in your Supabase project values:

```bash
cp .env.example .env
```

- `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` — Project Settings → API
- `DATABASE_URL` — the **Transaction pooler** connection string (port `6543`), used at runtime by Kysely
- `DIRECT_URL` — the **Direct** connection string (port `5432`), used by Prisma for migrations

### 2. Create the database schema

```bash
pnpm db:migrate     # creates the `profiles` table and the SQL migration
pnpm db:generate    # regenerate Kysely types after schema changes
```

> The `profiles` table mirrors Supabase `auth.users` (its `id` is the user id).
> Optionally add a trigger in Supabase to auto-insert a profile row on sign-up.

### 3. Run the app

```bash
pnpm dev
```

Open http://localhost:3000. Sign up at `/login`, then edit your profile on
`/dashboard` (a protected route).

## Project layout

```
src/
├─ app/
│  ├─ api/trpc/[trpc]/route.ts   # tRPC fetch handler
│  ├─ auth/callback/route.ts     # OAuth / magic-link code exchange
│  ├─ auth/signout/route.ts      # sign out
│  ├─ dashboard/page.tsx         # protected page (profile editor)
│  ├─ login/                     # email/password auth UI
│  ├─ layout.tsx                 # wraps app in TRPC + React Query provider
│  └─ page.tsx                   # home (server-side tRPC call)
├─ components/
│  ├─ profile-editor.tsx         # client component using api.* hooks
│  └─ site-header.tsx            # server component reading the auth user
├─ lib/supabase/
│  ├─ client.ts                  # browser Supabase client
│  ├─ server.ts                  # server Supabase client (cookies)
│  └─ middleware.ts              # session-refresh helper used by proxy.ts
├─ server/
│  ├─ db/
│  │  ├─ index.ts                # Kysely instance (pg Pool)
│  │  └─ types.ts                # generated by prisma-kysely
│  └─ trpc/
│     ├─ context.ts              # db + supabase + user
│     ├─ trpc.ts                 # public/protected procedures
│     ├─ root.ts                 # app router
│     └─ routers/profile.ts      # example router (Zod + Kysely)
├─ trpc/
│  ├─ react.tsx                  # client provider + api hooks
│  ├─ query-client.ts            # React Query + superjson
│  └─ server.ts                  # server-side caller (RSC)
└─ proxy.ts                      # Next.js 16 proxy (auth session refresh)
```

## Scripts

| Command            | Description                              |
| ------------------ | ---------------------------------------- |
| `pnpm dev`         | Start the dev server                     |
| `pnpm build`       | Production build                         |
| `pnpm typecheck`   | `tsc --noEmit`                           |
| `pnpm lint`        | ESLint                                   |
| `pnpm db:migrate`  | Create/apply a dev migration             |
| `pnpm db:generate` | Regenerate Kysely types from the schema  |
| `pnpm db:deploy`   | Apply migrations (CI/prod)               |
| `pnpm db:studio`   | Open Prisma Studio                       |
